Performance Labels

Automated product labels for Google Merchant Center and ad reporting.

1. Main Menu

  • Setup Script: checklist for platform credentials, label settings, Google Ads script, and auto-fetch status.
  • Google Ads Script [To Copy]: opens the script that must be copied into Google Ads Scripts.
  • Settings: admin tools for Run Now, label settings, platform settings, and auto-fetch setup.
  • Labels Documentation: explains generated label columns and thresholds.
  • Script Documentation: opens this guide.

2. First-Time Setup

  • Open Performance Labels -> Setup Script.
  • Choose your platform in Platform Settings: WooCommerce, Shopify, Gomag, or GA4.
  • Save the required credentials for the selected platform.
  • Open Label Settings and review timeframe, thresholds, and output labels.
  • Open Auto-Fetch Setup once to install the daily schedule for the active platform.

3. Platform Settings

  • WooCommerce: store domain, API key, and API secret.
  • Shopify: domain, client credentials, Product ID Matching, and Shopify Country Code.
  • Gomag: ApiShop, Apikey, Product ID Matching, parent-vs-variant catalog rows, optional GMC_Feed_2 ID mode, and optional order statuses to count. Leave statuses unchecked to count all orders.
  • GA4: GA4 Property ID only.

4. Auto-Fetch and Run Now

  • Settings -> Auto-Fetch Setup installs or repairs the active platform schedule.
  • Auto-fetch is single-platform: changing platform and running setup removes stale platform triggers.
  • Settings -> Run Now is for initial runs or testing. It starts the active platform and installs only the needed worker continuation trigger.
  • Store platforms sync products/orders and then recalculate labels. GA4 runs only the GA4 report.

5. Google Ads Script

  • Open Performance Labels -> Google Ads Script [To Copy].
  • Copy the code and paste it in Google Ads -> Tools & Settings -> Scripts.
  • Set the SPREADSHEET_URL variable to this spreadsheet URL.
  • Authorize, run once, then schedule it Daily at 4:00 AM, before the Sheets store auto-fetch.
Do not paste the Google Ads script into this Sheet Script Editor. It belongs only in Google Ads Scripts.

6. Generated Sheets

  • Overview: live status, sync logs, and error messages.
  • GAds: Google Ads metrics imported by the Google Ads script.
  • WooCommerce, Shopify, or Gomag: source store data.
  • Metrics: consolidated source data plus Google Ads metrics.
  • GMC_Feed: final supplemental feed with product IDs and labels.
  • GMC_Feed_2: optional secondary feed when prefix/suffix or a Gomag secondary ID mode is configured.
  • Product ID Matching controls the Default Feed ID and can optionally use a Google Ads-matching platform ID when that ID matches GAds.id.
  • Catalog completeness checks are logged in Overview; no separate audit sheet is required.

7. Central Logging

  • Optional central logging is enabled only when PL_CENTRAL_LOG_SHEET_URL is set in the script code and the executing account can write to that sheet.
  • It appends important setup, run, warning, error, and audit events to your private central Events sheet.

8. Publishing the Feed

  • In Google Sheets, open File -> Share -> Publish to web.
  • Select GMC_Feed and format Comma-separated values (.csv).
  • Use the published CSV URL as a supplemental feed in Google Merchant Center.
Note: API keys are stored in Google Script Properties, not in the spreadsheet cells.
